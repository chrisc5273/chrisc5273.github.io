Project: Cybersecurity Study Hub

This page outlines the plan for my ITIS 3135 project, following the assignment requirements.

Client and Project Category

  • Client Category: 01 - Peer Classmate. My client will be a small group of fellow students in my ITIS 3200 class.
  • Project Category: 3 - Student group at UNCC that encourages participation. The goal is to build a web application that provides a service for us (students) to learn a subject (cybersecurity).

1. Project Overview

  • Application and Purpose: The application is a "Cybersecurity Study Hub" for the ITIS 3200 (Intro to Info Security & Privacy) course. Its purpose is to consolidate and organize study materials, key concepts, and external resources into a single, easy-to-navigate website. It provides a "read-only" service to help students learn the subject, as our course does not cover data interaction involving CRUD.
  • Intended Users: The primary users are students currently enrolled in ITIS 3200, as well as the course TA who can provide content.
  • Overview of Content: The site will feature curated study guides (organized by module), a filterable glossary of key terms (like 'CIA Triad', 'SOP', 'SIEM', etc.), a visual gallery of important diagrams (OSI model, network topologies), and a page of external resources (practice sites, tools).

4. Site Map

The website will have 6 pages to ensure a clear separation of content. All pages will share a component header and footer. The site map is embedded below.

The 6 pages will be:

  1. `index.html` (Home)
  2. `guides.html` (Study Guides)
  3. `glossary.html` (Key Terms)
  4. `diagrams.html` (Diagram Gallery)
  5. `resources.html` (External Links)
  6. `contact.html` (Contact/About)

5. Page Design

Below is the detailed design for each page listed in the site map.

Page 1: `index.html` (Home)

  • Name: Home
  • Purpose: To introduce the Study Hub, state its purpose, and provide quick navigation to the main content areas.
  • Audience: First-time and returning students.
  • Content: Welcome text, a brief "About" paragraph, and three "Quick Link" cards (e.g., "View Guides", "Search Terms", "See Diagrams").
  • Data Entry: No.
  • Validations: N/A.
  • Buttons/Links: Main navigation, footer links, and the three Quick Link cards.
  • Actions: All links navigate to their respective pages.

Page 2: `guides.html` (Study Guides)

  • Name: Study Guides
  • Purpose: To provide detailed, topic-by-topic study guides in an organized, collapsible format.
  • Audience: Students preparing for exams or reviewing modules.
  • Content: A **jQuery-UI Accordion** widget. Each accordion header will be a module title (e.g., "Module 1: CIA Triad"). The content will be the study notes for that module.
  • Data Entry: No.
  • Validations: N/A.
  • Buttons/Links: The accordion headers themselves are the links/buttons.
  • Actions: Clicking a header will use JavaScript (from the jQuery-UI widget) to toggle the display of the content panel.

Page 3: `glossary.html` (Key Terms)

  • Name: Key Terms
  • Purpose: To provide a client-side, filterable A-Z list of all key terms and acronyms for the course.
  • Audience: Students who need to quickly look up a term.
  • Content: A search/filter input field and a definition list (`
    `) of all terms and definitions.
  • Data Entry: Yes, a filter input field (this is a read-only operation).
  • Validations: N/A.
  • Buttons/Links: No buttons.
  • Actions: As the user types in the filter box, JavaScript will run `onkeyup` to hide/show terms in the definition list that match the input.

Page 4: `diagrams.html` (Diagram Gallery)

  • Name: Diagram Gallery
  • Purpose: To provide a visual gallery of key security/networking diagrams (e.g., OSI Model, 3-way handshake).
  • Audience: Visual learners.
  • Content: A grid of thumbnail images. This will be implemented as an **interactive image gallery** (e.g., a lightbox).
  • Data Entry: No.
  • Validations: N/A.
  • Buttons/Links: The thumbnail images.
  • Actions: Clicking a thumbnail will use JavaScript to open a full-size modal/lightbox view of the image with a caption.

Page 5: `resources.html` (External Links)

  • Name: External Links
  • Purpose: To host a curated list of external resources (tools, practice sites, articles), loaded dynamically.
  • Audience: Students looking for more information or practice (e.g., Wireshark or subnetting practice).
  • Content: A container that will be populated with links. This page will use **AJAX**. The link data will be stored in a static `links.json` file.
  • Data Entry: No.
  • Validations: N/A.
  • Buttons/Links: Potentially category filter buttons (e.g., "Tools", "Practice", "Articles").
  • Actions: On page load, **AJAX** will fetch the `links.json` file and use JavaScript to parse it and render the links as HTML.

Page 6: `contact.html` (Contact/About)

  • Name: Contact/About
  • Purpose: To provide information about the client/study group and a simple contact form.
  • Audience: Students who want to contribute content or get in touch.
  • Content: "About the Group" text and a simple HTML form (Name, Email, Message).
  • Data Entry: Yes, the form fields.
  • Validations: Yes. JavaScript will be used to validate that the email is in a correct format and the message is not empty before "submitting."
  • Buttons/Links: Form "Submit" button.
  • Actions: Clicking "Submit" will trigger JS validation. If valid, the script will hide the form and display a "Thank you" message. **No data will be stored** (respecting the no-CRUD rule).

6. Dynamic Functionality on the Website

This website will implement several key pieces of JavaScript-driven dynamic functionality:

  1. jQuery-UI Accordion (`guides.html`):
    • Why: To present a large amount of study guide text in a compact, organized way.
    • Example URL: `https://jqueryui.com/accordion/`
  2. Live Search/Filter (`glossary.html`):
    • Why: To allow users to instantly find a specific term in a long list without page reloads.
    • Example URL: `https://www.w3schools.com/howto/howto_js_filter_list.asp`
  3. AJAX Content Loading (`resources.html`):
    • Why: To meet the project's AJAX requirement and to practice separating data (`.json`) from presentation (`.html`).
    • Example URL: `https://www.w3schools.com/js/js_ajax_intro.asp`
  4. Image Gallery/Lightbox (`diagrams.html`):
    • Why: To provide a clean, interactive way to view important diagrams in detail.
    • Example URL: `https://lokeshdhakar.com/projects/lightbox2/` (This is just an example of the *type* of interactivity).
  5. Form Validation (`contact.html`):
    • Why: To practice client-side validation and provide user feedback without a backend.
    • Example URL: `https://www.w3schools.com/js/js_form_validation.asp`
